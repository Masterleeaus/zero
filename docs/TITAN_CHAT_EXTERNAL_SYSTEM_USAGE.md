# TITAN CHAT — EXTERNAL SYSTEM USAGE

**Generated:** 2026-04-03  
**Purpose:** How to use the Titan AI chatbots in other systems and platforms.

---

## 1. Overview

The Titan Chat system can be embedded in or connected to external systems via:

1. **REST API** — `POST /api/titan/chat/send` (auth:sanctum)
2. **Embedded chatbot widget** — JavaScript snippet via the External-Chatbot extension
3. **Channel webhooks** — Messenger, WhatsApp, Telegram, Voice, Webchat

---

## 2. REST API — Chatbot in Other Systems

### Authentication
All API requests require a Bearer token (Laravel Sanctum).

```http
Authorization: Bearer <sanctum-token>
Content-Type: application/json
```

### Send a Chat Message

```http
POST /api/titan/chat/send
```

**Request body:**
```json
{
  "input": "What is the status of job #1234?",
  "surface": "workspace",
  "session_id": "optional-session-uuid",
  "company_id": 5,
  "model": "gpt-4o"
}
```

**Surface values:**
| Value | Description |
|-------|-------------|
| `workspace` | Generic workspace chat |
| `aichatpro` | AIChatPro advanced UI surface |
| `canvas` | Canvas structured drafting surface |
| `chatbot` | Chatbot widget surface |

**Response:**
```json
{
  "ok": true,
  "status": "ok",
  "runtime": "openai",
  "output": "Job #1234 is currently In Progress...",
  "decision": { ... },
  "tokens_used": 345
}
```

### Check Bridge Status

```http
GET /api/titan/chat/status
```

Response:
```json
{
  "bridge": "TitanChatBridge",
  "channels": ["messenger", "whatsapp", "telegram", "voice", "webchat", "external"],
  "pipeline": {
    "omni": "App\\TitanCore\\Omni\\OmniManager",
    "router": "App\\TitanCore\\Zero\\AI\\TitanAIRouter",
    "memory": "App\\Titan\\Core\\TitanMemoryService"
  }
}
```

## 3. Embedded Chatbot Widget

The External-Chatbot extension provides an embeddable JavaScript widget.

### Requirements
- External website must be on the allowed origins list
- Embed token issued per chatbot

### Embed snippet (example)
```html
<script src="https://yoursite.com/vendor/chatbot/widget.js"
        data-chatbot-id="abc123"
        data-embed-token="your-embed-token">
</script>
```

The widget communicates via the External channel adapter:
```
Widget message → ChatbotController webhook → ExternalChatbotChannelAdapter::toEnvelope()
              → TitanChatBridge::chatViaChannel('external', ...) → TitanAIRouter
```

### Features available in embedded context
- Full AI chat response
- Memory across sessions (by visitor_id + session_id)
- Persona-specific instructions
- Plan-based rate limiting

### Limits in embedded context
- No direct action execution (all actions require approval)
- Auth context is the chatbot owner's company_id
- Guest sessions (no user_id)

## 4. Channel Webhooks

### Messenger (Facebook)
**Webhook URL:** `POST /webhook/messenger/{chatbot_id}`  
**Extension:** ChatbotMessenger  
**Requirements:** Facebook App, Page Access Token in chatbot config

### WhatsApp (Twilio)
**Webhook URL:** `POST /webhook/whatsapp/{chatbot_id}`  
**Extension:** ChatbotWhatsapp  
**Requirements:** Twilio account, WhatsApp Business number

### Telegram
**Webhook URL:** `POST /webhook/telegram/{chatbot_id}`  
**Extension:** ChatbotTelegram  
**Requirements:** Telegram Bot token

### Voice (ElevenLabs)
**Flow:** Audio → Speech-to-text → `VoiceChannelAdapter::toEnvelope()` → TitanAIRouter → TTS synthesis  
**Extension:** ChatbotVoice + ElevenlabsVoiceChat  
**Requirements:** ElevenLabs API key, voice model selection

### Webchat
**Embed via:** Webchat extension widget  
**Requirements:** Widget configured per chatbot

## 5. Shared Runtime Services Required

All external integrations depend on:

| Service | Config | Required |
|---------|--------|---------|
| TitanAIRouter | `config/titan_core.php`, `config/titan_ai.php` | Yes |
| TitanMemoryService | `config/titan_memory.php` | Yes |
| AI provider (OpenAI/Bedrock) | `config/openai.php` / AWS config | Yes |
| TitanTokenBudget | `config/titan_budgets.php` | Yes |
| Sanctum auth | `config/sanctum.php` | For REST API |
| Queue worker | `config/queue.php` | For async signal processing |

## 6. Tenant / Auth Context

Every external chat request must include or resolve:

| Field | How set |
|-------|---------|
| `company_id` | From authenticated user (REST) OR from chatbot config (channel adapters) |
| `user_id` | From authenticated user OR null for guest |
| `session_id` | Caller-provided OR generated by adapter |

Middleware protecting REST API: `auth:sanctum`, `throttle:60,1`

---

## 7. Action Permissions in External Context

Chat-initiated actions (create job, send message, etc.) always pass through:

```
TitanAIRouter → SignalBridge → ApprovalChain
```

External systems never receive direct side-effect execution — they receive a `signal_id` that can be approved/rejected through the Titan admin.

**Signal approval endpoint:**  
`POST /api/titan/signal/callback` (requires `ValidateZylosSignature` middleware)

---

## 8. Rate Limits

| Context | Limit |
|---------|-------|
| REST API (`/api/titan/chat/send`) | 60 requests/minute per token |
| MCP API | 60 requests/minute |
| Signal callbacks | 120 requests/minute |
| Channel webhooks | Managed by channel extension middleware |

## 9. Example: Integrating in a Third-Party CRM

```php
// From an external PHP system using Guzzle:
$response = Http::withToken($titanApiToken)
    ->post('https://yourtitan.com/api/titan/chat/send', [
        'input'      => 'Draft a follow-up email for customer John Smith',
        'surface'    => 'workspace',
        'session_id' => 'crm-customer-john-smith-session',
    ]);

$output = $response->json('output');
// → "Dear John, following up on your recent enquiry..."
```

## 10. Example: Mobile App Integration (TitanCommand / TitanGo)

Mobile apps use the same REST API with Sanctum token auth:

```
POST /api/titan/chat/send
Authorization: Bearer <mobile-user-token>

{
    "surface": "workspace",
    "input": "Schedule a job for site #456 tomorrow at 9am",
    "session_id": "mobile-user-123-session"
}
```

PWA offline support: if offline, the PWA queues the request via `PwaDeferredReplayService` and replays it via `titan.chat.send` on reconnect.
