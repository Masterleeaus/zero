# Gaps and Assumptions in Current Docs

## Missing canonical docs
1. No single canonical doc that fully states **Omni owns comms** and **Nexus = Work / Money / Office / Grow hubs**.
2. No single canonical doc defining **Nexus Home as chat-first launcher** with 5 cards.
3. No dedicated doc that formalizes the **top 10 Nexus Spark mini-app priority list**.
4. No dedicated doc that consolidates **Scout configuration UI requirements**.
5. No dedicated doc that consolidates **Lifecycle wizard UI requirements**.
6. No dedicated doc that consolidates **Field-worker mobile panel requirements**.
7. No per-mini-app route/data/API contract spec.
8. No standard GitHub issue template for **Spark mini React app** creation.

## Working assumptions for issue generation
- Re-map old Jobs Mode material to **Work Hub**.
- Re-map old Finance Mode material to **Money Hub**.
- Re-map old Admin Mode material to **Office Hub**.
- Re-map old Social / growth-related material to **Grow Hub**.
- Re-map any Comms Mode material to **Omni**, not Nexus.
- Treat Spark outputs as standalone mini React apps that can also be embedded into the future Nexus shell.
- Keep mobile-first and PWA-friendly requirements explicit in every issue.
- Respect `DOC121_Node_vs_PWA_Data_Boundary.md`: PWA/mini-app state is not the source of operational truth.
