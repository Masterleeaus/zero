# DOC 26 — Rewind Compatibility Layer

All five modes must support rewind.

Rules:
1. Nothing is hard-deleted.
2. Original process is preserved.
3. Correction process is linked to original.
4. Downstream effects are analyzed.
5. Rollback cascades are tracked.
6. Replacements are audit-linked forever.

Mode examples:
- Jobs: wrong booking date corrected
- Comms: wrong recipient/message corrected
- Finance: wrong invoice/payment corrected
- Admin: wrong policy/compliance record corrected
- Social: wrong post/campaign execution corrected
